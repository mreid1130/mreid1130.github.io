Inlines, Blocks, and Inline-blocks

I can say pretty confidently that I'm not alone in having some difficulty totally understanding the display property's functionality when first learning CSS. The goal of this blog is to both reinforce my own knowledge and help others better understand how three of the most common display values, inline, block, and inline-block, behave when applied to HTML elements. An HTML file is made up of HTML elements. Every element contains content surrounded by a start and end tag (e.g. p, h1, img, a). And every piece of content can be surrounded by 3 layers, called the padding, border, and margin. I'll be referencing how these parts of an HTML element are affected by the three display values.



Inline

  • Does not insert line break before or after. If multiple inline elements are not seperated by any other type of elements, the inline elements will sit side by side.
  • Takes up as much width as necessary.
  • Does not accept top or bottom margin settings. Margins can be applied to the left or right side of the element, and padding can be applied to all four sides.
  • Can be inserted seamlessly into text without breaking up the paragraph or adding a line break.


Block

  • Can have a set height/width. If no width is set, a block will expand to fit the width of the parent container. If no height is set, a block will expand naturally to fit it's child elements.
  • Inserts a line break before and after the element.
  • Can have margin and padding.
  • Will automatically place itself below elements above it on an HTML file.


Inline-Block

Inline-Block is by far the most complicated of the three display properties, and one that I am still trying to wrap my head around. Inline-block is essentially a hybrid property of the, you guessed it, block and inline values.
  • Accepts both width and height elements.
  • Elements can appear next to each other like inline elements, but retain block elements like a set width and height
These are just a few of the many differences between three display property values. The differences are really much greater than my shallow overview would suggest, but hopefully some of these points will stick with you when deciding which display value to set to an element.
